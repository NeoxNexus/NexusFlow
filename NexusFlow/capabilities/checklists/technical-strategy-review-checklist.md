# 技术方案评审检查清单

## 目标检查
- [ ] 真实目标是否明确
- [ ] 评审对象是否明确
- [ ] 当前约束是否明确
- [ ] 输出要求是否明确

## 问题定义检查
- [ ] 是否区分了目标与实现路径
- [ ] 是否识别了核心矛盾
- [ ] 是否说明了现状与边界

## 方案判断检查
- [ ] 是否给出了推荐路线
- [ ] 是否区分了推荐项与备选项
- [ ] 是否说明为什么推荐
- [ ] 是否避免只罗列选项不收口

## MVP 检查
- [ ] 是否明确 MVP 范围
- [ ] 是否明确当前不做的事项
- [ ] 是否控制了复杂度

## 落地检查
- [ ] 是否给出阶段划分
- [ ] 是否给出执行顺序
- [ ] 是否指出风险与阻塞点
- [ ] 是否给出下一步建议

## 输出质量检查
- [ ] 是否结论优先
- [ ] 是否结构清晰
- [ ] 是否有边界意识
- [ ] 是否避免把未验证内容说成事实
