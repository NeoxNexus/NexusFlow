# 候选识别规则（初版）

## 目标
用最简单、可解释的规则，从工作记录中识别值得沉淀的候选任务。

## 入池建议规则
满足以下任意 2 条，可进入候选池：

1. 最近 2 周出现 >= 3 次
2. 单次耗时 >= 15 分钟
3. 输出结构相对稳定
4. 主要步骤具有重复性
5. 存在明显重复脑力劳动
6. 多次需要查相似资料 / 做相似判断 / 输出相似格式

## 优先级建议规则
优先级较高的候选通常满足：
- 高频
- 流程稳定
- 输出可模板化
- 节省时间明显
- 维护成本可控

## 暂不建议沉淀的任务
以下情况优先暂缓：
- 强依赖临场创意判断
- 每次输入和目标差异过大
- 很少发生
- 自动化失败代价高
- 维护成本明显高于收益

## 推荐形态判断
### 更偏 Skill
- 输入输出清晰
- 工具调用路径稳定
- 可复用执行

### 更偏 Prompt
- 更像分析框架
- 输出结构稳定但过程灵活

### 更偏 Script
- 更像固定数据处理或命令执行

### 更偏 Checklist
- 更关注人工检查与避免遗漏
