# 新项目启动 Prompt

## 适用场景

当需要启动一个新的项目、知识骨架、工作流系统或自动化仓库，并希望快速完成目标收口、文档落地、目录搭建和仓库初始化时使用。

适合场景：
- 新项目启动
- 新工作流仓库搭建
- 新知识系统骨架初始化
- 新自动化系统 MVP 建仓

不适合场景：
- 已有成熟项目上的小改动
- 单文件脚本临时实验
- 已有完整模板且无需重新设计的仓库

---

## 使用目标

输出一个最小正确的新项目启动方案，并完成基础骨架定义。

默认目标：
1. 明确项目目标与边界
2. 明确当前阶段应落的基础文件
3. 明确目录结构
4. 明确当前不做事项
5. 明确初始化与提交顺序

---

## 推荐输入

至少提供：
- 项目名称
- 项目目标
- 当前阶段目标
- 主要约束
- 是否需要 GitHub 仓库
- 是否需要数据层 / schema / 模板骨架

- 先明确目标，再落方案

6. 当前不建议做的事项
