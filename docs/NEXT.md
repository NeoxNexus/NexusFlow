# NEXT / NexusFlow 真实运行指南

## 当前结论

NexusFlow 现在已经具备最小运行条件。

接下来不要继续无上限建设，而应进入真实运行阶段：
- 记录真实工作
- 做周期回顾
- 维护候选池
- 观察能力资产是否真的有用

当前重点不是“再做更多”，而是“先用起来”。

- 项目仓库
- README
- 架构设计文档
- roadmap
- 实施计划
- 决策记录

这已经足够进入真实运行。

## 运行目标

真实运行阶段主要验证 4 件事：

1. 是否能持续记录关键工作样本
2. 是否能从样本中识别出重复任务候选
3. 已沉淀能力资产是否真的能节省时间、减少遗漏
4. 哪些候选值得继续推进，哪些应暂缓

---

## 日常使用方式

### 1. 遇到关键工作时，优先记日志
适合记录的工作包括：
- 技术方案评审
- 新项目启动
- 调研整理
- Bug 排查
- 结构化总结
- 明显重复的工程动作

记录位置：
- `NexusFlow/logs/manual-journals/`

建议做法：
- 不要求什么都记
- 优先记高价值任务
- 每天 1~3 条关键样本就够

---

### 2. 出现“这事好像经常做”时，补候选评估
当你明显感觉某类工作重复出现，或者已经有 2~3 个类似样本时：

记录位置：
- `NexusFlow/analysis/candidates/`

使用模板：
- `candidate-evaluation-template.md`

重点回答：
- 这事是不是高频
- 是否有稳定步骤
- 更像 Skill / Prompt / Script / Checklist 哪一种
- 值不值得进入实施队列

---

### 3. 每周做一次 weekly review
建议固定每周一次。

记录位置：
- `NexusFlow/logs/weekly-reviews/`

使用模板：
- `weekly-review-template.md`

每次至少看这几件事：
- 本周有哪些高频任务
- 新增了哪些候选
- 哪些候选应进入优先队列
- 已有能力资产是否真的有帮助
- 下周是否该继续推进某个候选

---

### 4. 使用已有能力资产，而不是只新增资产
当前已有：
- `technical-strategy-review`
- `project-bootstrap`

接下来优先：
- 真实使用
- 看效果
- 记反馈

而不是立刻继续造第三、第四个能力资产。

---

## 推荐节奏

### 每天
- 记录 1~3 条关键工作样本
- 如果出现明显重复任务，记下它的重复信号

### 每周
- 完成 1 次 weekly review
- 更新候选池优先级
- 记录已有能力资产反馈

### 每两周 / 每月
- 决定是否推进新的候选
- 决定是否补脚本层
- 决定是否把某个轻量资产升级为更正式能力

---

## 当前优先事项

### 优先做
1. 持续记样本
2. 维护 weekly review
3. 继续使用已有能力资产
4. 更新候选池

### 暂时不优先做
1. 自动化脚本扩张
2. 复杂评分系统
3. 自动识别算法
4. 多能力资产并行建设
5. 重 Skill 化

---

## 判断是否进入下一阶段

当满足以下条件时，可以考虑进入“半自动增强”阶段：

- 已积累 10+ 条有效样本
- 已完成至少 2 次 weekly review
- 已有 1~2 个能力资产被真实使用多次
- 已能看出哪些候选确实值得继续做

在此之前，不要急着补太多自动化。

---

## 一句话原则

> 先让 NexusFlow 在真实工作里跑起来，再决定该扩什么，而不是先把它做复杂。
