# 候选任务评估模板

- 候选名称：技术方案评审与执行顺序整理
- 任务描述：围绕一个新系统或新工作流目标，完成问题 framing、边界定义、风险识别、MVP 范围判断与阶段执行顺序输出。
- 典型触发场景：新项目启动、复杂工作流设计、方案收口、平台/系统搭建前评审。
- 关联事件 / 样本：
  - 2026-03-24-sample-01-strategy-review.md
- 最近出现频率：高（按用户工作类型判断）
- 单次平均耗时：60-120 分钟
- 主要重复步骤：
  1. 明确目标与边界
  2. 拆问题与分层
  3. 判断 MVP 与后续阶段
  4. 输出 roadmap 与建议顺序
- 当前痛点：每次都要重新搭建评审框架，容易遗漏风险、边界和不建议事项。
- 预期节省：提高收口速度，减少重复组织成本。
- 风险点：过度模板化会削弱具体场景判断。
- 推荐沉淀形态：
  - [ ] Skill
  - [x] Prompt
  - [ ] Script
  - [x] Checklist
  - [ ] 暂缓

## 评分
- 频率（1-5）：5
- 稳定性（1-5）：4
- 节省潜力（1-5）：5
- 维护风险（1-5）：2
- 综合判断：高优先级候选，适合优先沉淀为 Prompt + Checklist 组合。

## 决策
- 是否进入实施队列：
  - [x] 是
  - [ ] 否
- 如果进入，优先级：
  - [x] 高
  - [ ] 中
  - [ ] 低
- 最小实现范围：先做评审 Prompt 模板和配套 checklist，不急于做 Skill。
- 不做范围：不做自动化生成完整架构的全自动系统。
- 备注：这是当前最值得做的首批候选之一。
