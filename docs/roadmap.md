# NexusFlow / Roadmap

## 总体策略

以“单场景闭环优先”为路线，分阶段推进：
- 先搭骨架
- 再积累数据
- 再识别候选
- 再沉淀能力
- 最后扩展自动化

---

## Phase 0 / 基建（0-1 周）

### 目标
建立最小可运行骨架。

### 交付物
- 目录结构
- SQLite schema
- 关键设计文档
- 日志模板
- 周回顾模板
- 候选评估模板

### 完成标准
- 可以开始记录工作事件
- 可以初始化数据库
- 文档足以支撑后续 session 延续

---

## Phase 1 / 观察（1-3 周）

### 目标
积累真实样本，建立 baseline。

### 交付物
- 2 周工作日志
- 初版 weekly review
- Top10 高频任务统计
- 初版候选池

### 完成标准
- 至少形成 20-50 条有效事件记录
- 至少识别 3-5 个重复任务候选
- 可以初步判断候选形态

---

## Phase 2 / 固化（3-6 周）

### 目标
把 1 个高价值候选转化为示范能力资产。

### 交付物
- 1 个 Skill / Prompt / Script / Checklist MVP
- 上线检查清单
- 初版反馈记录

### 完成标准
- 至少有 1 个能力资产被真实使用
- 能够记录使用反馈
- 能评估是否值得继续维护

---

## Phase 3 / 自动识别增强（6-10 周）

### 目标
在数据和反馈基础上提高识别效率。

### 可选交付物
- 候选评分脚本
- 相似任务聚类实验
- 频率告警机制
- 半自动候选推荐

### 前提
- Phase 1 和 Phase 2 已验证有效
- 已有足够真实样本

---

## Phase 4 / 治理与扩展（10-16 周）

### 目标
建立长期治理能力。

### 可选交付物
- 能力资产分类
- 季度审计机制
- 下线/合并流程
- 跨任务联动能力

---

## 明确不做（当前阶段）

以下事项暂不进入当前 roadmap：
- 通用自动化平台建设
- 多用户/团队协作扩展
- 复杂 ML/向量数据库识别
- 多个能力资产并行孵化
- 自动更新与复杂版本治理

---

## 当前建议优先级

### 立即做
1. 文档固化
2. 模板固化
3. 开始记录样本

### 紧接着做
4. weekly review 机制
5. 候选池初版
6. 首个示范能力资产选择

### 后面再做
7. 数据处理脚本
8. 候选评分自动化
9. 自动识别增强
